# CASP LLM Verification Study

An automated evaluation pipeline to test LLM-generated ACSL specifications on [CASP](https://huggingface.co/datasets/nicher92/CASP_dataset) benchmark using Frama-C WP.

For each CASP benchmark pair:

- We generated C code annotated by different LLMs.
- Run Frama-C with Alt-Ergo on each version.
- Collect proof results _(proved goals, total goals, time, solver stats)_.
- Compare each LLM result against the reference verification (from casp).
- Aggregated everything into a **CSV** for statistical analysis.

Enables measurement of :
- How often each model fully verifies.
- How close each model is to the reference proof.
- How robust each model is across different programs.

### CSV Description

Evaluation pipeline produces a CSV file `(framac_results.csv)`. 
- **Rows :** Each row corresponds to the verification result of one LLM-generated specification _(related to one CASP pair)_.
- **Columns :** The table below describes the meaning of each column.

| Column                                | Description                                                                             |
| ------------------------------------- | --------------------------------------------------------------------------------------- |
| `directory`                           | CASP benchmark pair (folder name).                                    |
| `file`                                | Path to analyzed C file.                                                       |
| `filename`                            | Name of C file containing the LLM-generated specification.                          |
| `provider`                            | LLM provider used to generate the specification _(e.g., `openai`, `gemini`, `deepseek`)._ |
| `model`                               | LLM model used for generation.                                                 |
| `returncode`                          | Exit code returned by Frama-C _(`0` = success, `1` = error)._                             |
| `elapsed_s`                           | Total runtime of Frama-C verification _(seconds)._                                    |
| `date`                                | Timestamp when experiment was executed.                                             |
| `wp_goals_scheduled`                  | Nb. of proof obligations generated by Frama-C WP plugin.                         |
| `proved_goals`                        | Nb. of proof obligations successfully proved.                                        |
| `total_goals`                         | Total number of proof obligations generated.                                            |
| `timeout`                             | Nb. of goals that timed out during automated proof.                                  |
| `unreachable`                         | Nb. of goals identified as unreachable.                                              |
| `terminating`                         | Nb. of termination-related proof goals.                                              |
| `function`                            | Name of the analyzed C function.                                                        |
| `qed_count`                           | Nb. of goals solved by the internal Qed simplifier.                                  |
| `qed_t_min_ms`                        | Min. solving time for Qed _(ms)_.                                            |
| `qed_t_mid_ms`                        | Median solving time for Qed _(ms)_.                                             |
| `qed_t_max_ms`                        | Max. solving time for Qed _(ms)_.                                            |
| `alt_ergo_2_4_3_count`                | Nb. of goals handled by Alt-Ergo.                                         |
| `alt_ergo_2_4_3_t_min_ms`             | Min. solving time for Alt-Ergo _(ms)_.                                       |
| `alt_ergo_2_4_3_t_mid_ms`             | Median solving time for Alt-Ergo _(ms)_.                                        |
| `alt_ergo_2_4_3_t_max_ms`             | Max. solving time for Alt-Ergo _(ms)_.                                       |
| `meta_file_name`                      | Original CASP dataset source file.                                                      |
| `meta_verified`                       | Whether the reference dataset version verifies successfully.                            |
| `meta_verified_goals`                 | Nb. of proved goals in the reference version.                                        |
| `meta_total_goals`                    | Total number of goals in the reference version.                                         |
| `proved_ratio`                        | Ratio of proved goals (`proved_goals / total_goals`).                                   |
| `vs_meta_ratio`                       | Ratio of proved goals relative to the reference (`proved_goals / meta_total_goals`).    |
| `delta_proved_vs_meta_verified_goals` | Difference between proved goals and reference proved goals.                             |
| `delta_total_vs_meta_total_goals`     | Difference between total goals and reference total goals.                               |

### Reproduce Experiment

Following steps allow rebuilding dataset and regenerating final verification results _(framac_results.csv)_.

##### 01. Install dependencies

```python
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Ensure Frama-C _(with WP + Alt-Ergo)_ is installed :

```bash
frama-c -version
alt-ergo --version
```

##### 2. Sample CASP benchmark pairs

Download and sample a subset of the CASP dataset : `python 01_sample_casp.py`. This creates the directory structure:

```bash
casp_10/
   00_327/
   01_57/
   ...
   09_52/
```

Each folder contains:

- **code.c** : original CASP program,

- **spec.acsl** : reference specification,

- **meta.txt** : reference verification metadata,

##### 3. Generate LLM specifications

Generate ACSL using multiple LLM providers : `python 02_generate_acsl_specs.py`

This produces files such as:

- _code\_with\_llm\_spec\_openai\_gpt-5.2.c_
- _code\_with\_llm\_spec\_openai\_gpt-4o-mini.c_
- _code\_with\_llm\_spec\_gemini\_gemini-2.5-flash.c_
- _code\_with\_llm\_spec\_deepseek\_deepseek-chat.c_

##### 4. Run Frama-C verification

Execute Frama-C WP on each generated specification and collect results : `python 03_run_framac_collect.py`

For every CASP pair, the script:

- runs Frama-C with Alt-Ergo
- stores verification logs in each directory
- extracts proof statistics

##### Final output

The pipeline produces `framac_results.csv`. This file aggregates ***all verification results across models and CASP pairs***, 
_(statistical analysis, comparison of LLM-generation, etc)_.
